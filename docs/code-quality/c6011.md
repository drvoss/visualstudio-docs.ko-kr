# <a name="c6011"></a>C6011
경고 C6011: > \<name NULL 포인터를 역참조 하 고 있습니다.

 이 경고는 null 포인터를 역참조 하 고 있음을 나타냅니다. 포인터 값이 잘못된 경우 결과가 정의되지 않습니다.

## <a name="example"></a>예제
 다음 코드는 사용 가능한 메모리가 부족 한 경우 malloc 호출에서 null을 반환할 수 있기 때문에이 경고를 생성 합니다.

```
#include <malloc.h>

void f( )
{
  char *p = ( char * ) malloc( 10 );
  *p = '\0';

  // code ...
 free( p );
}
```

 이 경고를 해결 하려면 다음 코드에 나와 있는 것 처럼 null 값에 대 한 포인터를 검사 합니다.

```
#include <malloc.h>
void f( )
{
  char *p = ( char * )malloc ( 10 );
  if ( p )
  {
    *p = '\0';
    // code ...

    free( p );
  }
}
```

 매개 변수를 역참조 하기 전에 Pre 조건에서 Null 속성을 사용 하 여 매개 변수의 주석을 달고 있는 함수 내에 메모리를 할당 해야 합니다. 다음 코드는 먼저 메모리를 할당 하지 않고 함수 내에서 null 포인터 (`pc`)를 역참조 하려고 하기 때문에 경고를 생성 합니다 C6011.

```
#include <sal.h>
using namespace vc_attributes;
void f([Pre(Null=Yes)] char* pc)
{
  *pc='\0'; // warning C6011 - pc is null
  // code ...
}
```

 Malloc 및 free를 사용 하면 메모리 누수 및 예외 측면에서 많은 문제가 발생 합니다. 이러한 종류의 누수 및 예외 문제를 모두 방지 하려면 STL ( C++ 표준 템플릿 라이브러리)에서 제공 하는 메커니즘을 사용 합니다. 여기에는 [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)및 [vector](/cpp/standard-library/vector)가 포함 됩니다. 자세한 내용은 [스마트 포인터](/cpp/cpp/smart-pointers-modern-cpp) 및 [ C++ 표준 라이브러리](/cpp/standard-library/cpp-standard-library-reference)를 참조 하세요.
