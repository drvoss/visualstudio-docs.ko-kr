# <a name="c6331"></a>C6331
경고 C6331: 잘못 된 매개 변수: \<function >와 함께 MEM_RELEASE 및 MEM_DECOMMIT를 전달할 수 없습니다. 이로 인해이 호출이 실패 합니다.

 이 메시지는 잘못 된 매개 변수가 VirtualFree 또는 VirtualFreeEx에 전달 되 고 있음을 나타냅니다. VirtualFree 및 VirtualFreeEx 모두 플래그 (MEM_RELEASE &#124; MEM_DECOMMIT)를 조합 하 여 거부 합니다. 따라서 MEM_DECOMMIT 및 MEM_RELEASE 값은 동일한 호출에서 함께 사용할 수 없습니다.

 커밋을 해제할 때 및 릴리스가 독립적인 단계로 발생 하는 경우에는 필요 하지 않습니다. 커밋된 메모리를 해제 하면 페이지도 커밋을 해제할 때 됩니다. 또한이 함수의 반환 값이 무시 되지 않도록 해야 합니다.

## <a name="example"></a>예제
 다음 샘플 코드에서는이 경고를 생성 합니다.

```
#include <windows.h>
#define PAGELIMIT 80

DWORD dwPages = 0;  // count of pages
DWORD dwPageSize;   // page size

VOID fd( VOID )
{
  LPVOID lpvBase;            // base address of the test memory
  BOOL bSuccess;
  SYSTEM_INFO sSysInfo;      // system information

  GetSystemInfo( &sSysInfo );
  dwPageSize = sSysInfo.dwPageSize;

  // Reserve pages in the process's virtual address space
  lpvBase = VirtualAlloc (
                       NULL,                 // system selects address
                       PAGELIMIT*dwPageSize, // size of allocation
                       MEM_RESERVE,
                       PAGE_NOACCESS );
  if (lpvBase)
  {
    // code to access memory
  }
  else
  {
    return;
  }
  bSuccess = VirtualFree(lpvBase,
                0,
                MEM_DECOMMIT | MEM_RELEASE); // warning
  // code...
}
```

 이 경고를 해결 하려면 다음 코드와 같이 MEM_DECOMMIT 값을 VirtualFree 호출에 전달 하지 마십시오.

```
#include <windows.h>
#define PAGELIMIT 80

DWORD dwPages = 0;  // count of pages
DWORD dwPageSize;   // page size

VOID f( VOID )
{
  LPVOID lpvBase;            // base address of the test memory
  BOOL bSuccess;
  SYSTEM_INFO sSysInfo;      // system information

  GetSystemInfo( &sSysInfo );
  dwPageSize = sSysInfo.dwPageSize;

  // Reserve pages in the process's virtual address space
  lpvBase = VirtualAlloc (
                       NULL,                 // system selects address
                       PAGELIMIT*dwPageSize, // size of allocation
                       MEM_RESERVE,
                       PAGE_NOACCESS );
  if (lpvBase)
  {
    // code to access memory
  }
  else
  {
    return;
  }
  bSuccess = VirtualFree(lpvBase, 0, MEM_RELEASE);
  // code...
}
```

 Malloc 및 free (및 관련 동적 메모리 할당 Api)를 사용 하는 경우 메모리 누수 및 예외 측면에서 많은 문제가 발생 합니다. 이러한 종류의 누수 및 예외 문제를 모두 방지 하려면 STL ( C++ 표준 템플릿 라이브러리)에서 제공 하는 메커니즘을 사용 합니다. 여기에는 [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)및 [vector](/cpp/standard-library/vector)가 포함 됩니다. 자세한 내용은 [스마트 포인터](/cpp/cpp/smart-pointers-modern-cpp) 및 [ C++ 표준 라이브러리](/cpp/standard-library/cpp-standard-library-reference)를 참조 하세요.
