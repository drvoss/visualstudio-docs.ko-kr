# <a name="test-area-3-check-outundo-checkout"></a>테스트 영역 3: 체크 아웃/체크 아웃 실행 취소
이 소스 제어 플러그 인 테스트 영역에서는 **체크 아웃** 및 **체크 아웃 취소** 명령을 통해 버전 저장소에서 항목을 편집 하 고 되돌리는 작업을 다룹니다.

**체크 아웃**: 버전 저장소의 항목을 체크 아웃 한 것으로 표시 하 고 로컬 복사본을 읽기/쓰기로 수정 합니다.

체크 **아웃 취소**: 버전 저장소의 항목을 체크 인 한 것으로 표시 하 고, 체크 아웃 하기 전에 로컬 복사본을 상태로 되돌립니다 (옵션에 따라 다름).

## <a name="command-menu-access"></a>명령 메뉴 액세스

다음 [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] 통합 개발 환경 메뉴 경로가 테스트 사례에 사용 됩니다.

##### <a name="check-out"></a>체크 아웃:

- **파일**, **소스 제어**, **체크 아웃**합니다.

- **파일**을 **체크 아웃**합니다.

- 바로 가기 메뉴에서 **체크 아웃**합니다.

- 체크 아웃 취소: **파일**, **소스 제어**, **체크 아웃 실행 취소**

## <a name="common-expected-behavior"></a>일반적인 예상 동작

- 체크 아웃 작업 후에는 대상 파일 및/또는 폴더가 버전 저장소에서 체크 아웃 된 것으로 표시 됩니다.

- 버전 저장소는 올바른 사용자에 게 체크 아웃 하는 특성을 갖습니다.

- 사용자 설정에 따라 체크 아웃의 시간 및 날짜가 올바릅니다.

## <a name="test-cases"></a>테스트 사례

체크 아웃/체크 아웃 취소 테스트 영역에 대 한 특정 테스트 사례는 다음과 같습니다.

|------------|----------------|--------------------------------|

### <a name="case-3b-disconnected-check-out"></a>사례 3b: 오프 라인 체크 아웃

오프 라인 모드에서 작동 하면 버전 저장소에 직접 연결 되지 않은 경우 사용자가 계속 해 서 소스 제어를 지원할 수 있습니다. 이 작업은 등록 된 솔루션 및 프로젝트에 대 한 모든 관련 정보를 로컬로 캐시 하 여 수행 됩니다.

단독 체크 아웃 작업은 원본 제어 저장소에 연결 되어 있는 경우에만 발생할 수 있습니다. 공유 체크 아웃 작업은 언제 든 지 연결 됨 또는 연결 끊김 인지 여부에 관계 없이 발생할 수 있습니다. 따라서 버전 저장소에서 연결이 끊어진 경우에는 COS ( **체크 아웃 공유** ) 명령만 사용할 수 있습니다. 사용자가 변경한 내용을 바꾸기 위해 이전 버전을 검색할 수 없기 때문에 연결이 끊어져 있으면 **체크 아웃 취소가** 사용 하지 않도록 설정 됩니다.

사용자가 버전 저장소에 다시 연결 하면 등록 된 모든 솔루션 및 프로젝트의 체크 아웃 상태가 동기화 됩니다. 이렇게 하면 사용자가 수행한 체크 아웃에 대해 저장소에 필요한 업데이트가 수행 됩니다. 동기화가 수행 되 면 사용자는 정상적으로 (연결 된) 작업을 계속할 수 있습니다.

#### <a name="expected-behavior"></a>예상 동작

- 버전 저장소에서 분리 된 상태에서 **단독으로 체크 아웃** 명령을 사용할 수 없습니다.

- 버전 저장소에서 분리 된 상태에서 **체크 아웃 취소** 명령을 사용할 수 없습니다.

- **공유 체크 아웃** 명령이 작동 합니다.

|작업|테스트 단계|확인 결과가 예상 됩니다.|
|------------|----------------|--------------------------------|
|연결이 끊어진 상태에서 파일을 체크 아웃 한 다음 동기화를 위해 연결 합니다.|1. 소스 제어 변경 대화 상자 (**파일**, **소스 제어**, **소스 제어 변경**)를 사용 하 여 제어 된 프로젝트의 연결을 끊습니다.<br />2. 파일을 체크 아웃 합니다.<br />3. 경고 대화 상자에서 체크 아웃 (연결 끊김)을 클릭 합니다.<br />4. 파일을 편집 합니다.<br />5. 소스 제어 변경 대화 상자를 사용 하 여 연결 합니다.<br />6. 편집한 파일의 최신 버전을 가져옵니다.|일반적인 예상 동작|

### <a name="case-3c-query-editquery-save-qeqs"></a>사례 3c: 쿼리 편집/쿼리 저장 (QEQS)
 원본 제어에 있는 항목은 사용자가 파일을 쉽게 관리할 수 있도록 편집, 변경 및 저장에 대해 추적 됩니다. "체크 인" 된 제어 항목을 편집 하는 경우 QEQS는 시도한 편집을 가로채 고 파일을 편집 하기 위해 파일을 체크 아웃 하려고 하는지 사용자에 게 요청 합니다. **도구**, **옵션** 설정에 따라 사용자는 편집 하기 위해 파일을 체크 아웃 해야 합니다. 그렇지 않으면 메모리에서 복사본을 편집 하 여 나중에 체크 아웃할 수 있습니다. 사용자의 **도구**, **옵션** 설정이 체크 아웃 대화 상자를 표시 하 고 체크 아웃 하도록 설정 되지 않은 경우 사용자가 편집 하는 경우 가능 하면 파일이 자동으로 체크 아웃 됩니다.

#### <a name="expected-behavior"></a>예상 동작

- 체크 아웃 작업 후에는 대상 파일 및/또는 폴더가 버전 저장소에서 체크 아웃 된 것으로 표시 됩니다.

- 버전 저장소에는 올바른 사용자에 대 한 체크 아웃 특성이 있습니다.

- 체크 아웃의 시간 및 날짜가 올바릅니다 (사용자 설정에 따라).

- 대상 파일 또는 폴더의 로컬 복사본에 쓸 수 있습니다.

|작업|테스트 단계|확인 결과가 예상 됩니다.|
|------------|----------------|--------------------------------|
|체크 인 한 텍스트 파일 편집|1. 텍스트 파일을 포함 하는 새 프로젝트를 만듭니다.<br />2. 솔루션을 소스 제어에 추가 합니다.<br />3. **도구**, **옵션**, **소스 제어**, **디스크에서 읽기 전용으로 파일을 편집할 수 있도록 허용** 을 선택 하지 않은 상태로 설정 합니다.<br />4. **도구**, **옵션**, **소스 제어**를 설정 하 고 체크 인 한 파일을 **편집할 때** 콤보 상자에서 **체크 아웃 하 라는 메시지를 표시** 합니다.<br />5. **선택 된 파일을 저장할 때** 콤보 상자에서 **도구**, **옵션**, **소스 제어**, **체크 아웃 확인 메시지** 를 설정 합니다.<br />6. 편집기에서 텍스트 파일을 열고 파일에 새 텍스트를 입력 해 봅니다. 이 단계가 성공 하면 다음 단계를 계속 합니다.<br />7. **편집 하기 위해 체크 아웃** 대화 상자에서 **취소** 를 클릭 합니다. 이 단계가 성공 하면 다음 단계를 계속 합니다.<br />8. **도구**, **옵션**, **소스 제어**, **디스크에서 읽기 전용으로 파일을 편집할 수 있도록 허용** 을 선택 된 상태로 설정 합니다.<br />9. 편집기에서 프로젝트 파일을 열고 파일에 새 텍스트를 입력 해 봅니다. 이 단계가 성공 하면 다음 단계를 계속 합니다.<br />10. **편집 하기 위해 체크 아웃** 대화 상자에서 **편집** 을 클릭 합니다. 이 단계가 성공 하면 다음 단계를 계속 합니다.<br />11. 텍스트 파일을 편집 하 고 저장 하려고 합니다.|`Result of step 6:`<br /><br /> 편집 하기 위해 체크 아웃 대화 상자가 나타납니다.<br /><br /> `Result of step 7:`<br /><br /> 파일이 변경 되지 않았습니다.<br /><br /> `Result of step 9:`<br /><br /> 편집 하기 위해 체크 아웃 대화 상자가 나타납니다.<br /><br /> `Result of step 10:`<br /><br /> 메모리에서 프로젝트 파일을 편집할 수 있습니다.<br /><br /> `Result of step 11:`<br /><br /> 저장할 때 저장 시 체크 아웃 대화 상자가 나타납니다.|
|체크 인 된 솔루션 파일 편집|이전 테스트에 설명 된 단계를 반복 하지만 텍스트 파일을 수정 하는 대신 솔루션 속성을 변경 하 여 솔루션을 수정 합니다.|이전 테스트와 동일|
|체크 인 된 프로젝트 파일 편집|이전 테스트에 설명 된 단계를 반복 하지만 텍스트 파일을 수정 하는 대신 프로젝트 속성을 변경 하 여 프로젝트를 수정 합니다.|이전 테스트와 동일 합니다.|

### <a name="case-3d-silent-check-out"></a>사례 3d: 자동 체크 아웃
 이 하위 영역에서는 사용자의 **도구**, **옵션**, **소스 제어 설정**에 따라 **체크 아웃** 대화 상자가 나타나지 않는 체크 아웃 시나리오에 대해 설명 합니다.

#### <a name="expected-behavior"></a>예상 동작

- 체크 아웃 작업 후에는 대상 파일 및/또는 폴더가 버전 저장소에서 체크 아웃 된 것으로 표시 됩니다.

- 버전 저장소에는 올바른 사용자에 대 한 체크 아웃 특성이 있습니다.

- 체크 아웃의 시간 및 날짜가 올바릅니다 (사용자 설정에 따라).

- 대상 파일 또는 폴더의 로컬 복사본에 쓸 수 있습니다.

|작업|테스트 단계|확인 결과가 예상 됩니다.|
|------------|----------------|--------------------------------|
|파일에 대 한 자동 체크 아웃|1. **편집 시 파일을 자동으로 체크 아웃**하는 **도구**, **옵션**, **소스 제어** 를 설정 합니다.<br />2. 파일을 사용 하 여 새 프로젝트를 만듭니다.<br />3. 솔루션을 소스 제어에 추가 합니다.<br />4. 파일을 체크 아웃 합니다.|파일이 자동으로 체크 아웃 됩니다 (UI 없음).|
|프로젝트에 대 한 자동 체크 아웃|1. **편집 시 파일을 자동으로 체크 아웃**하는 **도구**, **옵션**, **소스 제어** 를 설정 합니다.<br />2. 새 프로젝트를 만듭니다.<br />3. 솔루션을 소스 제어에 추가 합니다.<br />4. 프로젝트를 확인 합니다.|파일이 자동으로 체크 아웃 됩니다 (UI 없음).|

### <a name="case-3e-undo-check-out"></a>사례 3e: 체크 아웃 실행 취소
 **체크 아웃 실행 취소** 는 파일의 체크 아웃 상태를 취소 하 고 파일에 대 한 변경 내용 체크 인을 방지 하는 데 사용 됩니다.

#### <a name="expected-behavior"></a>예상 동작

- 기본값은 사용자의 **체크 아웃 로컬 버전** 설정에 따라 달라 집니다. 사용자가 로컬 버전을 체크 아웃 하도록 선택한 경우 체크 아웃 취소의 기본값은 항상 체크 아웃 한 버전으로 되돌려야 합니다.

- 실행 취소가 승인 되 면 영향을 받는 파일에 대해 **솔루션 탐색기** 의 아이콘이 업데이트 되 고 **보류 중인 체크 인** 창에서 항목이 제거 됩니다.

|작업|테스트 단계|확인 결과가 예상 됩니다.|
|------------|----------------|--------------------------------|
|단독으로 체크 아웃 한 단일 파일의 체크 아웃 취소|1. 클라이언트 프로젝트를 만듭니다.<br />2. 솔루션을 소스 제어에 추가 합니다.<br />3. 파일을 독점적으로 체크 아웃 합니다.<br />4. 파일을 수정 합니다.<br />5. 체크 아웃을 실행 취소 합니다 (**파일**, **소스 제어**, **체크 아웃 취소**).|일반적인 예상 동작입니다.|
|공유를 체크 아웃 한 단일 파일의 체크 아웃 취소|1. 클라이언트 프로젝트를 만듭니다.<br />2. 솔루션을 소스 제어에 추가 합니다.<br />3. 공유 된 파일을 체크 아웃 합니다.<br />4. 파일을 수정 합니다.<br />5. 체크 아웃을 실행 취소 합니다 (**파일**, **소스 제어**, **체크 아웃 취소**).|일반적인 예상 동작입니다.|
|프로젝트에 파일을 추가한 후 프로젝트 체크 아웃 실행 취소|1. 새 프로젝트를 만들고 소스 제어에 추가 합니다.<br />2. 프로젝트를 체크 아웃 합니다.<br />3. 프로젝트에 파일을 추가 합니다.<br />4. 프로젝트의 체크 아웃을 실행 취소 합니다.|추가 된 파일이 솔루션 탐색기의 프로젝트에서 제거 됩니다.<br /><br /> 프로젝트가 더 이상 체크 아웃 되지 않습니다.|
|프로젝트에서 파일을 삭제 한 후 프로젝트 체크 아웃 실행 취소|1. 새 프로젝트를 만들고 소스 제어에 추가 합니다.<br />2. 프로젝트를 체크 아웃 합니다.<br />3. 프로젝트에서 파일을 삭제 합니다.<br />4. 프로젝트의 체크 아웃을 실행 취소 합니다.|삭제 된 파일은 솔루션 탐색기의 프로젝트 아래에 나타납니다.<br /><br /> 프로젝트가 더 이상 체크 아웃 되지 않습니다.|
